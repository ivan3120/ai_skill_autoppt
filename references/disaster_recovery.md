# 容灾技术知识

## 容灾等级

### 等级1: 数据级容灾
- 定期备份数据
- 恢复时间: 小时级
- 数据丢失: 天级

### 等级2: 应用级容灾
- 备用系统standby
- 恢复时间: 分钟级
- 数据丢失: 分钟级

### 等级3: 网络级容灾
- 多活架构
- 恢复时间: 秒级
- 数据丢失: 零

## 容灾组

### 概念
容灾组（Disaster Recovery Group）是由多个网元组成的容灾单元，当主用节点故障时，自动切换到备用节点。

### 类型

#### 1. 主备容灾
- 一主一备
- 自动/手动切换
- 适用场景: 核心网元

#### 2. 双主容灾
- 双活同时工作
- 负载均衡
- 适用场景: UPF等

#### 3. 多站点容灾
- 跨站点部署
- 地理级保护
- 适用场景: 关键业务

## 容灾指标

### RTO (Recovery Time Objective)
恢复时间目标，系统故障后恢复业务的最长时间。

### RPO (Recovery Point Objective)
恢复点目标，数据恢复的最大可接受丢失量。

### 可用性计算

```
可用性 = MTBF / (MTBF + MTTR) * 100%

其中:
- MTBF: Mean Time Between Failures (平均故障间隔)
- MTTR: Mean Time To Repair (平均修复时间)
```

## 容灾评估要点

1. **切换能力**: 是否支持自动切换
2. **数据一致性**: 切换后数据是否一致
3. **切换时间**: 切换需要多长时间
4. **业务影响**: 切换对业务的影响
5. **验证测试**: 是否定期进行容灾演练
