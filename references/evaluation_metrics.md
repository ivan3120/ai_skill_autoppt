# 评估指标说明

## 评估维度总览

| 维度 | 名称 | 权重 | 说明 |
|------|------|------|------|
| MT0 | 网元高稳 | 20% | 评估网元稳定性 |
| MT1 | 部署离散度 | 15% | 评估部署分散程度 |
| MT2 | 组网架构高可用 | 20% | 评估网络冗余设计 |
| MT3 | 业务路由高可用 | 15% | 评估路由可靠性 |
| MT4 | 网络容量均衡度 | 10% | 评估负载均衡 |
| MT5 | 网元版本EOS | 10% | 评估版本生命周期 |
| MT6 | 云平台版本EOS | 10% | 评估平台版本 |

## 评估等级

### 风险等级

| 等级 | 分数 | 颜色 | 说明 |
|------|------|------|------|
| 低 | 90-100 | 绿色 | 符合标准，无需关注 |
| 中 | 70-89 | 黄色 | 存在轻微风险 |
| 高 | 50-69 | 橙色 | 存在较大风险 |
| 严重 | <50 | 红色 | 存在严重风险 |

### 评估结果含义

- **通过**: 风险等级为"低"
- **警告**: 风险等级为"中"或"高"
- **不通过**: 风险等级为"严重"

## 计算方法

### MT0: 网元高稳

```
得分 = (1 - 告警网元数 / 总网元数) * 100
```

### MT1: 部署离散度

```
得分 = (同产品域最大部署站点数 / 总站点数) * 100
```

### MT2: 组网架构高可用

```
得分 = (有冗余的网元对数 / 总网元对数) * 100
```

### MT3: 业务路由高可用

```
得分 = (有备份路由的业务数 / 总业务数) * 100
```

### MT4: 网络容量均衡度

```
得分 = 100 - (CV * 50)
其中CV为变异系数
```

### MT5/MT6: 版本EOS

```
得分 = (在支持版本的网元数 / 总网元数) * 100
```
